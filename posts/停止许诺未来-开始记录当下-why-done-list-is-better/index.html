停止许诺未来，开始记录当下：Why done list is better

2025-12-19
philosophy

我相信每个人都曾陷入过“ 生产力焦虑 ”：觉得自己效率低下，于是疯狂钻研各种计划工具，试图把未来每一分钟都填满。但你是否质疑过一件事—— 做 To-Do List （待办列表）这件事，本身就一定是正确的吗？

1. 大脑是为“当下”设计的，而非“未来” 当我们写下 To-Do 时，我们是在向自己许诺一个未来。然而，人类的大脑在进化中是面向“当下”设计的。这方面的著作很多，例如《当下的力量》、心理学上的延迟满足、甚至于冥想本质也是引导你关注当下。当你沉迷于编写计划，你的大脑会产生一种“我已经做了”的错觉，而在执行时，由于它总是盯着明天，反而忽略了现在该做什么。

2. 压力与成就感的错位 To-Do List 的逻辑很奇怪：它最好的状态竟然是“列表为空”（分析到这里的时候我有点没绷住）。这意味着你每写下一项，都是在增加一份“未完成”的压力（负债）；你只有做完了，压力才会消失。 而 Done List 则相反：它没有预设的压力。写下它是最后一步， 记录就是一种“收获” 。Done List 是越多越好，回看它时，那种充实的成就感是待办清单永远无法提供的。

3. 它是你最真实的一面镜子 To-Do List 无法帮你认识自己，因为它往往包含了很多“激进的幻想”。而 Done List 是你行为的忠实记载。通过它，你能看清自己实际的工作量，识别出哪些是真正的产出，哪些是无效的挣扎。它甚至能沉淀知识——你在已办中记录的“坑”、心得甚至是吐槽，都是未来最宝贵的经验。

4. 帮助你总结汇报 Done List 是你行为的忠实记载，它能极大降低复盘和汇报时的心理压力。当你需要向他人汇报工作进度时，你不需要费力去验证 To-Do 列表上哪些真正做完了，Done List 本身就是一份现成的、现结的成果清单。也是说， Done-list真的可以帮助你减少工作量。 而To-do其实是给你新增了一份写todo的工作量。

如何写Done-list

想要真正发挥 Done-list 的威力，我建议在文档中建立一个简单的四栏表格。这不仅仅是为了整齐，更核心的逻辑在于通过 视觉布局 来重塑你的 心理权重 。

为什么这样排列？ 1. 日期： 这是表格的第一列。它强制你建立时间感。阅读已办列表的时候，一定要数一数日期。通常你会吓到的。无论正面或者负面。

2. 已办（核心主列）： 这是你成就感的来源，是你今天全部智慧和劳动的证明。你要把绝大部分注意力放在这里，通过“增加行”的方式不断 Append（追加），感受列表不断变长的满足感。你可以在这里 写任何东西，比如说吐槽和骂娘。 最完美的已办就是流水账。

3. 备注： 紧贴在“已办”右侧。在这里记录下那些你认为有价值沉淀的东西，或者自己的思考，等等。我最常见的是将冗长的命令，文档，操作手册写在这里。这些东西从已办中分离出来，方便你日后查找。这一列通常比待办要宽，因为各种操作命令通常贼他妈长……但是你的主要注意力应该放在已办上。

4. 待办： 这一部分用于记录那些你在干活的过程中意识到自己需要做的事情。比如说解决一个Bug的时候又冒了3个出来。你只能一个一个解决，那么选择一个去解决，剩下的两个就可以记在这里。它被刻意地放在边缘，一是避免压力，二是让你更容易速查。

我在实践过程中通常会将一些小的待办事项直接写到已办的吐槽里面（例如XX坏了，我靠……查了半天原来我需要先去yy搞定zz，再去aa搞定bb）。这样每天回顾的时候，自然而然你就知道自己有啥没干了。每天读读自己的历史也挺好玩的。只有那些重要的，值得被单拎出来的事件才会放到专门的待办里面去。

QA——关于 Done List 的一切疑虑

Q：如果我这一天或者这一周彻底搞砸了，什么都没干，也要记吗？

A： “什么都没干”和“干了但失败了”是两码事。对于后者我想，你会自己选择记下来的，因为饱含了经验。但如果是前者，随便了。通常，你在隔了一段时间打开已办列表之后，看到日期那一栏停留在很久之前，自己就会被震撼到，察觉到自己有所不对了。

Q：如果我能量真的很低，连打开列表的力气都没有，怎么办？

A： 不用强求。所有的生产力工具都是为你能量充足时准备的。如果你长时间能量低下、无法行动，这可能不是懒惰，而是生病了（比如抑郁）。这时候你需要的是休息、求助医生，而不是强行把自己拉起来。

Q：面对断更很久后的空白日期，我会感到强烈的羞耻感，不敢面对，怎么办？

A： 这就是 Done List 优于 To-Do List 的地方。如果你没做，打开 To-Do 看到的是堆积如山的债务，压力巨大；但打开 Done List，你面对的只是一段“空白”。只要点一下“增加一行”，随便吐槽一句：“我靠，这么多天没写，我真服了。” 这样你就重新开始了。补足空白的动力，远比还债的压力要轻松得多。

Q：Done List 里的那些吐槽、心得和怪癖，需要定期做“二次整理”吗？

A： 没必要。任何流程都有代价，二次整理的代价就是消耗你的精力和闲暇。我们改写 Done List 的初衷就是“少搞点事情”。这些吐槽提供了最真实的上下文，保留了你踩坑时的第一现场。如果文件太大了，把它归档到另一个文件里就行。把它当成服务器日志， 不要随随便便删，也不要随便整理，真实的历史才有价值。
